Manual Operativo: Estudio LPF

Gestión de actualizaciones, arquitectura de datos y administración de múltiples usuarios en la nube.

Propósito de la Guía

Esta plataforma centraliza los procedimientos técnicos del sistema de gestión jurídica. Incluye las instrucciones de actualización vía GitHub Pages, el análisis de almacenamiento de datos y el nuevo protocolo para la creación y gestión de múltiples usuarios, permitiendo que el Estudio LPF escale de una herramienta individual a un entorno colaborativo profesional.

1. Procedimiento de Actualización

Siga estos pasos para desplegar nuevas versiones del código en su repositorio.

📋

Copiar el Código Fuente

Seleccione y copie todo el contenido del bloque de código generado por la IA (archivo .html).

✏️

Editar en GitHub

Busque su archivo en el repositorio, pulse el ícono del lápiz, borre el contenido previo y pegue el nuevo código.

💾

Confirmar Commit

Finalice pulsando el botón verde "Commit changes" dos veces para que los cambios se apliquen a su URL pública.

2. Sistema Multi-Usuario

Con la integración de la Nube, el sistema permitirá la creación de perfiles personalizados para cada integrante del estudio.

👥 Niveles de Acceso

¿Cómo crear más usuarios?

1

Habilitar Auth: Se activará el módulo de autenticación de Firebase en el código.

2

Registro: Se habilitará una pantalla de "Registro" donde cada abogado ingresa su email y contraseña propia.

3

Base de Datos: Cada expediente guardado quedará vinculado al ID del usuario que lo creó, o podrá marcarse como "Público" para el resto del estudio.

👤

Perfil: Administrador

Capacidad para crear, editar y eliminar usuarios. Acceso total a la base de datos de expedientes de todos los miembros.

👨‍⚖️

Perfil: Colaborador

Gestión de expedientes propios y visualización de expedientes compartidos. Sin acceso a la configuración del sistema.

Estado Actual del Sistema:

Actualmente el sistema usa un **usuario estático local**. La creación de nuevos usuarios requiere la **Conexión a la Nube** para que las contraseñas sean validadas de forma segura.

3. Análisis de Datos & Almacenamiento

Comparativa técnica entre el almacenamiento local actual y la migración a la nube.

LocalStorage (Actual)

Ideal para: Pruebas y uso individual.

✗ No permite compartir datos con otros usuarios.

Cloud Firebase (Propuesto)

Ideal para: El Estudio LPF completo.

✓ Habilita el registro de múltiples usuarios y sincronización.
